# 프로젝트 가이드라인:

## 1. 개요
- **목적:** 선 잇기 2D 퍼즐
- **엔진/언어:** Typescript
- **핵심 목표:** 동작 가능한 웹앱
- **게임 룰:** 같은 색상의 점(노드)들을 터치/드래그하여 선으로 연결해야 함.
- **핵심 제약사항:** 서로 다른 연결 선이 **절대 교차해서는 안 됨**.
- **맵 구조:** 원형 또는 특정 바운더리 내에 점들이 배치되며, 모든 같은 색상의 짝을 선이 겹치지 않게 연결하면 스테이지 클리어.
- 선을 그어 연결을 완성하거나 스테이지 클리어 시 이펙트 & 소리 있음.
- 막혔을 때 사용할 수 있는 힌트 기능 존재.

## 2. 작업 규칙
- 모든 변경은: 설계 → 테스트 작성 → 구현 → 검증 순서로 진행
- 작업 완료 후 history.md에 한글로 이력 기록
- readme.md는 한글로 유지
- 한 번에 하나의 이슈만 해결하고 테스트 통과 확인 후 git push 후 다음 아이템으로
- 관리 에이전트가 개발, 검증, 디자인 에이전트 관리
- 작업 완료 후 github 푸시
  
## 3. 개발 주의 사항
- 실행시 외부 라이브러리 참조가 있으면 안됨
- HTML5 Canvas 기반으로 구현
- `python -m http.server 8001` 로 동작 가능해야 함

## 4. 이미지, 사운드 리소스
- 이모지를 최대한 사용
- 인터넷 무료 이미지 리소스 사용시 출처 표기
- 인터넷 무료 오디오 리소스 사용시 출처 표기 
- 오디오 리소스는 base64로 바꾸어 js에 포함 시킬 것

## 5. 코딩 규칙 (Coding Rules)
- 모든 코드는 모듈화하여 작성 (scene, player, board, ui 등 분리)
- 게임 루프는 고정된 프레임레이트(60 FPS)로 설정
- 변수명은 camelCase, 클래스명은 PascalCase 사용
- 주석은 코드의 의도를 명확히 설명할 것

## 6. 시스템/메커니즘 (Game Mechanics)
- **데이터**: JSON 파일로 레벨(1~100 스테이지의 점 위치, 색상, 맵 형태 등) 데이터 관리
- **DB**: 브라우저 IndexedDB 사용
- 이어하기 지원

## 7. 구조 (Directory Structure)
- `/assets`: 이미지, 사운드 파일
- `/src`: 소스 코드
- `/data`: 레벨 설정 파일
- `/tests`: 기능 테스트

## 8. 상세 지침 (Directives)
- 코드를 수정하기 전에 `test` 폴더의 기존 로직을 확인하라.
- 선 그리기(Path) 연산 및 충돌(교차) 판정 시 성능에 미치는 영향을 고려하여 최적화된 코드를 작성하라.
- DB 손상, 오류 시에도 게임은 동작해야 함
- PC 브라우저 동작 가능해야 함
- 모바일 터치 UI(드래그로 선 긋기) 완벽 지원
- 폴더7 펼침 화면, 닫힌 화면 모두 지원
- 폴더7 펼침 화면에서 Canvas가 고해상도로 선명하게 보이도록 할 것
- 게임은 1-100 레벨 맵을 지원해야 함
- 맵은 JSON 형식으로 수정과 추가가 쉬워야 함
- 사용자의 진행 상황은 Browser의 IndexedDB에 저장해서 이어하기 가능해야 함
- 다시 하기를 할 경우 사용자 컨펌 후 진행해야 함

## 9. 마무리
- 빌드시 `dist.js` 하나의 산출물로 나오도록 하기
- 실행에 필요한 파일만 `release` 폴더로 복사하는 `build.sh` 를 만들 것
- 실행에 필요한 파일만 `release` 폴더로 복사하는 Window cmd 용 `build.bat` 를 만들 것
